Bibilography Manager


Helps researchers in collecting and organizing literature.

ABOUT BIBILOGRAPHY MANAGER

Bibliography Manager is a tool specially designed for the researcher for collecting and organizing literature. The application is capable of storing a metadata about the research paper, books, journals and other publications. Researcher can create their own libraries which will contain metadata about different references. Also, researchers can share their libraries with other researcher with specific permissions.

  • Target users: Academic scholars and research professionals
  • Timeframe:Feburary 2019 - April 2019

CHALLENGES

  • A visitor to the website must be able to register as a user. The email address must be validated before registration is complete. When a user registers two libraries ('Trash' and 'Unfiled') must be created automatically.
  • A user must be able to update their registered details
  • A user must log in to the website to view the references in the active library.
  • A user must be able to logout from the website.
  • A user must be able to create a new library to store their references.
  • A user must be able to change the active library.
  • A user must be able to rename a library 'Trash' and 'Unfiled' must not be able to be renamed.
  • A user must be able to delete a library. The system must prompt the user asking where references (stored in the library to be deleted) should be moved to. 'Trash' and 'Unfiled' must not be deleted.
  • A user must be able to add references to a library.
  • A user must be able to update all fields for a reference in a library.
  • . A user must be able to delete both a single reference and a group of references A deleted reference must be moved to 'Trash'
  • A user must be able to move one or more references at a time from one library to another.
  • A user must be able to empty their 'Trash' library.
  • A user that owns a library must be able to share that library with other registered users. A user cannot share a library that the user does not own (i.e. has not created). A user cannot share a library that the user owns with a non-registered user.
  • . If a user owns a library then they must be able to see who that library is shared with. A user must not be able to see who a library is shared with if they do not own it.
  • The owner of a shared library must be able to unshare it.
  • . The user must be allowed to sort the references in both ascending and descending order by: title, author, year and key.
  • A user must be able to search for references that match criteria for: author, title and year. The search must allow for one or more libraries to be included in the search.
  • Inputs must be validated on the client and server side. Prevent SQL injection attacks etc.

DESIGN AND DEVELOPMENT KIT

DESIGN PROCESS

User research: We, as a team, conducted user research by asking some of our colleagues and students at the college to use similar web tools that are used for citing references for academic and research work. Apart from predefined requirements for the project mentioned under challenges section in this article, we made sure that we address the main problem that the users are facing using similar tools, and we want to make sure we solve that problem in our website.

Conceptualization and User flow: From conducting user interviews on the existing research and academic reference tools, we were able to analyse the data and created two personas. One is for a student, and other is for a researcher. After discussing how the two different personas use the website we are about to develop, we came up with a user mental model which helped us develop a rough user flow before we created a paper wireframe.

User Interface design and development :
Due to the tight schedule, we first made a priority that we meet all the requirements of the client. So, first, we created a fully functional prototype of our website and conducted usability testing on one student who seemed to be a better match for both of our personas. We took a brief note during usability testing and analysed the results. We immediately made the required changes to our design, and the usability of our site improved.
